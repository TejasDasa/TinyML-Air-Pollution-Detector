# Air Pollution Detector – ESP32 Firmware

TinyML wearable device that helps people with respiratory ailments navigate campus and home routes by continuously monitoring air quality and estimating respiratory distress risk.

## Hardware

| Component | Part | I²C Address |
|-----------|------|-------------|
| MCU | ESP32 DevKit | – |
| Environmental | BME688 | 0x76 (SDO→GND) |
| Particulate + VOC/NOx | SEN50-SDN-T | 0x69 (fixed) |
| CO₂ | SCD40-D | 0x62 (fixed) |

**Wiring** – all sensors share one I²C bus:

```
ESP32 GPIO 21 (SDA) ──── SDA of all 3 sensors
ESP32 GPIO 22 (SCL) ──── SCL of all 3 sensors
ESP32 3.3 V ──────────── VDD of all 3 sensors
ESP32 GND ────────────── GND of all 3 sensors
```

> The BME688 SDO pin must be connected to GND to set address 0x76. If you have a conflict on the bus, pull SDO high to use 0x77 and update `BME68X_I2C_ADDR_LOW` → `BME68X_I2C_ADDR_HIGH` in `main.cpp`.

## Firmware Architecture

```
src/
  main.cpp                 – Setup / loop orchestration
  sensors/
    sensor_data.h          – Shared SensorReading struct
    bme688.{h,cpp}         – BME688 + BSEC2 IAQ driver
    sen50.{h,cpp}          – SEN50 PM + VOC/NOx driver
    scd40.{h,cpp}          – SCD40 CO₂ driver
  ml/
    model_data.h           – TFLite flatbuffer (generated by training script)
    inference.{h,cpp}      – TFLite Micro inference + heuristic fallback
  ble/
    ble_service.{h,cpp}    – BLE GATT server (notify + command)
  pollution/
    aqi_calculator.{h,cpp} – US EPA AQI composite calculation
    exposure_tracker.{h,cpp}– Session exposure accumulator + waypoint log
training/
  train_model.py           – TF training → TFLite export → C header
  requirements.txt         – Python dependencies
```

## Getting Started

### 1. Install PlatformIO

```bash
pip install platformio
# or use the VS Code PlatformIO extension
```

### 2. Build and flash

```bash
cd Air-Pollution-Detector
pio run --target upload
pio device monitor --baud 115200
```

The device starts in **heuristic scoring mode** until a trained model is deployed.

### 3. Train a model (optional but recommended)

```bash
cd training
pip install -r requirements.txt
python train_model.py
# → outputs training/output/model_data.h
cp training/output/model_data.h src/ml/model_data.h
pio run --target upload
```

Set `MODEL_LOADED 1` is already done automatically by the script. After reflashing, Serial will confirm `TFLite ML` inference mode.

To collect real training data, walk your routes with the device logging to serial and capture with any serial logger. Save as `training/data/collected_data.csv` with columns: `pm2_5, pm10, voc_index, nox_index, co2_ppm, iaq, temperature, humidity, pressure, aqs_label, rdr_label`.

## BLE Protocol

Connect to device named **AirGuard** using any BLE scanner or a custom phone app.

| Characteristic | UUID suffix | Property | Format |
|----------------|-------------|----------|--------|
| Live data | `...26a8` | Notify | Binary `LivePacket` struct (see `ble_service.h`) |
| Session summary | `...26a9` | Read + Notify | JSON string |
| Control | `...26aa` | Write | JSON command |

**Control commands** (write as UTF-8 string):

```json
{"cmd":"reset_session"}    // clear exposure accumulator
{"cmd":"save_bme_state"}   // force-save BSEC2 calibration to NVS
{"cmd":"status"}           // print status to Serial
```

## Output Fields

| Field | Range | Description |
|-------|-------|-------------|
| `aqs` | 0–1 | Air Quality Score (1 = clean, 0 = very polluted) |
| `rdr` | 0–1 | Respiratory Distress Risk (1 = high risk) |
| `aqi_composite` | 0–500 | US EPA AQI scale composite |
| `cumulative_exposure` | 0–∞ | Session exposure integral (AQI × seconds) |

### AQI Categories

| AQI | Category | Colour |
|-----|----------|--------|
| 0–50 | Good | Green |
| 51–100 | Moderate | Yellow |
| 101–150 | Unhealthy for Sensitive Groups | Orange |
| 151–200 | Unhealthy | Red |
| 201–300 | Very Unhealthy | Purple |
| 301–500 | Hazardous | Maroon |

## BSEC2 Calibration

The BME688 IAQ accuracy goes through 4 levels (0–3). The device logs `[WAIT] Sensors stabilising...` until accuracy ≥ 1. Full calibration (accuracy = 3) can take up to 30 minutes of continuous operation. Calibration state is saved to NVS every 10 minutes and restored on reboot.
